# 资源夜市一条街 - CLAUDE.md

## 项目定位

资源夜市一条街（renmaiquan.top）是一个面向中文互联网的**商业人脉资源对接平台**，用3D地球视觉+搜索+内容分发的方式，把有资源的人、找资源的人、群组织、项目发起人、引荐人和协同人连接起来，通过"审核、脱敏、会员解锁、邮箱触达、人工桥接、风控保护"的方式，完成资源发现、需求匹配与合规协同。

**网站定位**：个人经验交流与信息分享平台，内容仅代表作者个人观点，不构成投资建议或商业承诺。

---

## 技术栈

- **后端**: Express.js + Sequelize ORM + MySQL
- **前端**: 原生 HTML/CSS/JS（无框架）
- **认证**: JWT + bcrypt 密码加密

## 居间护航模块

### 为什么要做这个模块

很多人有这样的心态：**帮忙牵个线、搭个桥，觉得是举手之劳，不好在钱上开口。**

但事实是：能帮你找到对的人、能说清楚你的需求、能推动事情落地——这本身就是一种**信息资产**，是多年人脉积累的变现能力。

平台设计这个模块，就是为了解决两个问题：

1. **好人不吃亏**：你帮忙引荐了，利益要保障，不能白忙一场
2. **信息资产被保护**：你的关系链不被泄露，跳单绕开你

### 对外名称
**居间护航**（对外不直接用"居间"等敏感词）

### 核心业务流程
```
用户A问到我 → A注册 → A填项目信息
       ↓
A的朋友是拍板人？
       ↓              ↓
    是            否
     ↓              ↓
A是【一级引荐人】 A是【二级引荐人】
     ↓              ↓
A发链接给朋友B  A发链接给朋友B
       ↓              ↓
   B是拍板人？    B是拍板人？
       ↓              ↓
     ...            ...

最终：找到一级引荐人2人 → 链条成立 → 开始交流
```

### 操作节点引导提示语

进入每个操作节点时，旁边显示提示语：

| 操作节点 | 提示语 |
|---------|--------|
| 进入居间护航首页 | "双一级居间人共拿60%、二级居间人分40%，规则透明无克扣" |
| 点击"创建项目" | "请明确总居间费金额，系统将自动按60%、40%拆分两级分配池" |
| 一级引荐人点击"添加二级" | "您分配的金额将从40%的二级池里扣除，不会占用您的60%核心收益" |

### 角色定义（3问自测）

**为什么要做3问自测**：帮忙牵线搭桥是有价值的事，好人不该白忙，信息资产需要保护。通过3个问题，用户可以清楚地知道自己能扮演什么角色、应得多少回报。

**3问引导式设计（每问都有3个引导提示）**：
- 💡 为什么要问这个（作用）
- 🎯 回答后你会知道什么（结果）
- ✨ 建议怎么填（正向鼓励）

**每问回答后都有正向鼓励语**：
- 问题1选"是" → "👍 好的！你是资源的源头方！"
- 问题1选"否" → "🌟 没问题！帮忙引荐也是一种能力！"
- 问题2选"是" → "💪 你能直接推动事情进展！"
- 问题2选"否" → "🤝 没关系！通过朋友牵线也很有价值！"
- 问题3选"是" → "🎉 很好！引荐成功率高！"
- 问题3选"否" → "✨ 即使深一度关系，引荐也有价值！"

**三问逻辑**：

| 问题 | 选项 | 结论 |
|------|------|------|
| 1. 你是总拍板人吗？ | 是 → | 你是甲乙其中一方的**拍板人** |
| | 否 → | 进入问题2 |
| 2. 你能直接与拍板人沟通、做他的代言人吗？ | 可以 → | 你是**一级引荐人** |
| | 不行 → | 进入问题3 |
| 3. 你的朋友（中间人）是拍板人吗？ | 是 → | 你是**二级引荐人**（深一度） |
| | 不是 → | 你是**二级引荐人**（深二度） |

**底部真诚提醒**：⚠️ 真诚是保护你名声的护城河，你的朋友链条很多人都能看到，请慎重、真实地选择。

**角色说明：**

| 角色 | 说明 | 可见信息 |
|------|------|----------|
| 拍板人 | 最终决策人，属于甲乙双方之一端 | 项目简报、进度状态 |
| 一级引荐人（0-1人/岸） | 离拍板人最近，能直接推动进展 | 另一侧中间人数量（不知人名） |
| 二级引荐人（0-N人/岸） | 只负责引荐，不参与协同对接 | 全部中间人数量 |

### 分润规则（明牌）
```
居间费总计 = 100%

一级引荐人（0-1人/岸）
• 离甲方最近的那个人
• 离乙方最近的那个人
每岸合计30%，两岸合计60%
（因为承担最多协同对接工作）

二级引荐人（一级之后的所有人）
共40%，每人 40% ÷ 实际人数
（只负责引荐，不参与协同对接）
```

### U型链路可视化（动态）

**核心特点**：人数不固定，根据实际加入的人动态显示。

**动态变化**：
- 一级引荐人：每岸可能 0-1 人
- 二级引荐人：每岸可能 0-N 人
- U型曲线高度随人数自适应
- 分润比例实时计算显示

**可视化元素**：
- 左岸：一方（拍板人+一级+多个二级）
- 右岸：另一方（拍板人+一级+多个二级）
- 中间：居间费分配比例（60%一级 + 40%二级）
- 实时显示链条状态

**没有一级时显示空状态**："等待一级引荐"

### 明牌规则

**禁止行为**：
- ❌ 跳单（直接联系对方拍板人）
- ❌ 瞎传话（擅自修改/隐瞒信息）
- ❌ 泄露信息（向不该知道的人透露）

**正确行为**：
- ✅ 明牌（所有信息通过平台传递）
- ✅ 留痕（所有沟通记录可查）
- ✅ 按规则分配

## 重要原则

1. **中性表达**: 平台不代表任何特定公司，以中立身份运营
2. **合规优先**: 所有功能和表达必须合规、克制、正气
3. **脱敏展示**: 公开信息默认脱敏，保护资源方和平台
4. **审核前置**: 所有内容先审核再展示
5. **免费高效**: 融资评估主打免费+2分钟出结果
6. **明牌规则**: 居间护航打明牌，按规则来
