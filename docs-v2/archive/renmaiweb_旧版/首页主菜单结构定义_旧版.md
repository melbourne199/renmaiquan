# 首页主菜单结构定义

## 背景说明

首页主菜单是用户进入平台后的第一层导航，需与后台数据模型、页面路由完全对齐。本次梳理旨在统一命名、理清边界，避免"资源星河"等历史遗留名称继续干扰产品定位。

**重要参考**：已有小程序"交个朋友点个赞"在线上运行，截图显示的实际菜单结构与本定义一致。

---

## 七个一级主菜单（与小程序一致）

| 序号 | 菜单名称 | 小程序对应 | 说明 |
|------|----------|------------|------|
| 1 | 首页 | 首页tab | 3D地球视觉入口，保留现有交互 |
| 2 | 行业群码 | 分类Tab | 行业群码聚合，含二维码展示 |
| 3 | 政企资源 | 分类Tab | 机关人脉 + 企业人脉 |
| 4 | 最新求助 | 分类Tab | 用户发布的求助/需求信息 |
| 5 | 民企金融 | 分类Tab | 金融业务模块 |
| 6 | 居间程序 | 分类Tab | 居间业务留痕与协同推进 |
| 7 | 行业动态 | 分类Tab | 警示案例，后台可添加发布 |

---

## 各菜单详细说明

### 1. 首页（地球页）

- **定位**：平台视觉入口 + 品牌第一印象 + 分发到各模块
- **保留内容**：3D地球、中国聚焦、城市点位、城市交互
- **品牌统一**：对外统一称"资源夜市"，不再使用"资源星河"
- **改造方向**：顶部文案和导航与下方7个一级菜单对齐
- **小程序显示**：底部Tab导航，图标+文字

### 2. 行业群码

- **定位**：低门槛入口产品，最容易传播和产生用户动作
- **数据模型**：Group
- **核心规则**：
  - 入群二维码（7天有效期）
  - 群主二维码（备用联系方式）
  - 形成双通道结构，保证长期可用
- **分类维度**：行业 + 地区
- **审核机制**：先审后发

### 3. 政企资源

- **定位**：核心资源层，区分"机关人脉"和"企业人脉"
- **数据模型**：
  - GovernmentResource（机关人脉）
  - EnterpriseLike（企业人脉）
- **人员信息格式**（根据小程序截图）：

  **机关人脉**：
  ```
  区域 + 部门简称 + 序号（可选）
  例如：广州市环保0年（实际显示：广州市环保0年）
       广州市环保1号
       广州市环保2号
  ```
  序号规则：
  - 0年 = 该部门的资深人员/主要领导
  - 1号 = 该部门第一负责人
  - 2号 = 该部门第二负责人
  - 以此类推

  **企业人脉**：
  ```
  地区 + 企业简称 + 部门 + 岗位
  例如：深圳腾讯CSIG产品总监
       （深圳 = 地区，腾讯 = 企业简称，CSIG = 部门，总监 = 岗位）
  ```

- **分类维度**：地区 + 行业/部门 + 岗位级别
- **小程序数据**：显示 "政企资源 36个"

### 4. 最新求助

- **定位**：用户主动发起的支援请求，调动平台资源响应
- **数据模型**：HelpRequest
- **信息格式**：
  ```
  地区 + XX部 + XX岗位
  例如：找广州教育部3号
       （表示需要广州市教育部相关资源/帮助）
  ```
- **核心字段**：地区、目标部门/岗位、悬赏金额（可选）、有效期
- **小程序数据**：显示 "最新求助 11个"

### 5. 民企金融

- **定位**：金融业务专项入口
- **数据模型**：Project
- **业务方向**：企业融资、借贷、供应链金融等民企金融需求
- **区别说明**：与"政企资源"的政府端、企业端人脉不同，本模块专注金融业务本身
- **小程序数据**：显示 "民企金融 3个"

### 6. 居间程序

- **定位**：为居间人/引荐人提供的协同推进与留痕工具
- **数据模型**：Referral（扩展）
- **核心能力**：
  - 发起协同链
  - 逐级邀请参与方
  - 记录关键节点
  - 保留过程痕迹
  - 角色边界识别（甲方拍板人、乙方拍板人、一级引荐人、二级引荐人）
- **对外表达**：合作推进护航，不直接用"居间"等敏感词

### 7. 行业动态

- **定位**：信任建设 + 风险提示
- **数据模型**：Case（成功案例）、Pitfall（避坑指南）
- **内容来源**：后台可添加，运营团队维护
- **作用**：
  - 成功案例 → 证明平台有结果
  - 避坑指南 → 证明平台有风控意识

## 需同步修改的代码位置

### public/index.html

当前问题：
- 品牌名："资源星河"（应为"资源夜市"）
- 导航项：群码聚合、能量信息、求助信息、行业动态（与新定义不对应）

需修改：
- navbar-brand → "资源夜市"
- 导航项 → 对应7个一级菜单

### server/models/index.js

当前模型基本匹配新结构：
- Group → 行业群码 ✓
- EnterpriseLike → 企业人脉 ✓
- GovernmentResource → 机关人脉 ✓
- HelpRequest → 最新求助 ✓
- Project → 民企金融 ✓
- Referral → 居间程序（扩展）✓
- 案例/避坑 → 需新增模型

---

## 优先级建议

1. **第一优先**：修改 public/index.html 的品牌名和导航，与7个一级菜单对齐
2. **第二优先**：确认各菜单对应的数据模型是否完整
3. **第三优先**：开发"居间程序"模块
4. **第四优先**：补充案例/避坑的数据模型
