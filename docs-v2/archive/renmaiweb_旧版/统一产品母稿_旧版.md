# 资源夜市一条街统一产品母稿

## 0. 文档定位

这份文档是当前阶段的统一产品母稿，用来合并以下几类信息，并作为后续首页设计、平台开发、会员设计、后台设计、推广文案、短视频导流和 SEO/AI 推广的共同依据。

- 既有核心方案文档
- 当前项目代码所体现出的实际进展
- 新补充的首页方向
- 新补充的真实业务逻辑
- 新补充的居间 / 引荐 / 桥接逻辑

从这一版开始，产品理解不再拆成“旧文档一套”“新增备注一套”“首页另算一套”，而是统一按本母稿执行。

## 1. 项目一句话定义

`资源夜市一条街` 是一个面向中文互联网传播场景的资源入口平台，用短视频、搜索和中文内容把有资源的人、找资源的人、群组织、项目发起人、引荐人和协同人连接起来，通过“审核、脱敏、会员解锁、邮箱触达、人工桥接、风控保护”的方式，完成资源发现、需求匹配与合规协同。

## 2. 平台核心定位

这个网站不是单纯的资料站，也不是纯二维码站，更不是炫技型视觉站。

它的真实定位是：

`资源入口页 + 信任建立页 + 内容分发页 + 合规桥接平台`

它要解决的是中文互联网里一类长期真实存在、但表达上又必须克制的问题：

- 很多人想找群
- 很多人想找组织
- 很多人想找靠谱的人脉入口
- 很多人有项目、有资源、有渠道，但没法公开讲太明白
- 很多人既想保护资源边界，又想让合作发生
- 很多人处在中间协同位置，但缺乏一个合规、体面、可留痕的工具

因此，平台的外在表达必须正气、克制、中文化、合规；平台的内在逻辑则要充分理解中国式资源连接、引荐、桥接、协同的真实需求。

## 3. 平台总业务闭环

平台的总体闭环可以概括为：

`短视频 / 搜索流量进入 -> 首页建立信任 -> 免费浏览脱敏摘要 -> 注册 / 会员升级 -> 解锁更深层信息 -> 邮箱触达 / 人工桥接 -> 项目协同推进 -> 留痕与风控保护`

这意味着平台不是靠一页落地页完成价值，而是靠多层级承接：

1. 内容传播层
2. 首页承接层
3. 资源浏览层
4. 会员解锁层
5. 邮箱与人工桥接层
6. 居间与协同层
7. 风控与履约边界层

## 4. 首页总纲

## 4.1 首页定位

首页必须同时承担三类任务：

- 资源入口页
- 信任建立页
- 内容分发页

首页不是：

- 炫技展示页
- 纯宇宙概念页
- 只讲情绪、不讲业务的视觉页

## 4.2 3D 地球的角色

首页必须保留 3D 地球。

但地球的角色必须被定义清楚：

- 地球负责高级感
- 地球负责格局感
- 地球负责技术感
- 地球负责建立“这不是小打小闹站点”的第一印象

地球不负责独立完成转化。

真正负责转化的是：

- 业务入口
- 首页文案
- 会员体系
- 信任说明
- 审核与脱敏规则
- 联系与桥接机制

也就是说：

`地球是气场，业务结构才是成交入口。`

## 4.3 首页应传达的核心认知

用户进入首页后，应在很短时间内明白：

- 这是什么网站
- 这里能找到什么
- 这里为什么值得信
- 我下一步该点哪里
- 深层信息为什么不是随便公开的

因此首页必须清楚表达：

- 这里有群码 / 社群 / 联系入口
- 这里有政府 / 企业 / 项目 / 人脉 / 需求信息
- 内容先审核、再展示
- 公开页默认脱敏
- 更深层信息通过会员解锁、邮箱触达或人工桥接
- 平台提供合规的居间与协同服务能力

## 4.4 首页结构原则

首页的结构建议按以下优先级建立：

1. 品牌与一句话价值
2. 3D 地球主视觉
3. 核心业务入口
4. 信任与规则说明
5. 会员解锁逻辑
6. 内容分发模块
7. 居间 / 桥接能力说明

## 5. 真实业务逻辑

## 5.1 平台服务的不是单一角色，而是三类人

平台至少服务三类核心用户：

### A. 资源发布者

这类人发布的是自己能联系到、能触达到、能搭上线的对象，例如：

- 自己能联系到的政府资源
- 自己能联系到的企业资源
- 自己能联系到的有实力的人
- 自己所在或管理的社群 / 群组织

### B. 需求发布者

这类人发布的是自己正在找的人或资源，通常会带有明确筛选条件，例如：

- 想找某地区的人
- 想找某部门的人
- 想找某行业的人
- 想找某岗位的人
- 想找某类项目协同对象

### C. 居间 / 引荐 / 协同人

这类人不是单纯发内容，也不是单纯看内容，而是帮助连接发生的人。他们的核心诉求是：

- 能接住资源
- 能接住需求
- 能把人桥接起来
- 能保留自己的过程贡献
- 能在合规边界内推进合作

## 5.2 平台承载的真实内容类型

平台的真实内容不应再被拆散理解，而要统一看成“资源供给 + 资源需求 + 社群入口 + 协同服务”四层。

### 第一层：资源供给

- 我能联系到的政府资源
- 我能联系到的企业资源
- 我能联系到的关键人物
- 我能提供的行业社群 / 二维码入口

### 第二层：资源需求

- 我正在找谁
- 我正在找哪个地区的资源
- 我正在找哪个部门的资源
- 我正在找哪个行业 / 岗位 / 圈层的人

### 第三层：社群入口

- 行业群
- 项目群
- 地区群
- 资源协作群
- 交流组织入口

### 第四层：协同与桥接

- 引荐
- 桥接
- 邮箱触达
- 人工对接
- 项目协同护航

## 6. 二维码 / 社群聚合总纲

## 6.1 为什么群码功能必须保留

群码功能是平台最适合短视频导流、最容易触发真实动作、最容易形成传播的入口功能之一。

很多用户进入平台后的第一需求不是“研究复杂服务”，而是：

- 找群
- 进组织
- 找圈层
- 找同行

所以群码 / 社群聚合不是配角，而是低门槛入口产品。

## 6.2 群码模块的真实设计逻辑

群码模块不是简单上传一张二维码，而是要解决社群入口长期不可持续的问题。

微信群二维码有 7 天有效期，这是业务设计中必须正视的现实规则。

因此，发布社群类内容时应遵守以下原则：

1. 不能只上传 7 天效期的入群二维码
2. 发布者必须上传群主本人二维码，或稳定联系入口
3. 当入群码失效后，用户仍能通过群主码继续建立联系
4. 平台要把“入群码”和“群主联系入口”区分展示

## 6.3 可上传的联系入口类型

平台应支持的入口形态包括：

- 入群二维码
- 群主二维码
- 个人联系二维码
- 社群入口说明
- 邮箱入口
- 备用联系入口

这部分不是单纯技术上传功能，而是平台最关键的“资源入口结构”之一。

## 6.4 群码模块的对外表达边界

群码模块对外必须强调：

- 找组织
- 找圈层
- 找真实交流入口
- 找行业协作入口

不应强调：

- 野路子导流
- 不透明拉人
- 任何灰色聚合暗示

## 7. 审核、脱敏、会员、邮箱、人工桥接的统一逻辑

## 7.1 统一原则

平台所有核心内容默认走以下逻辑：

`先审核 -> 再展示 -> 公开页脱敏 -> 深层信息分级触达`

这条规则适用于：

- 政府资源
- 企业资源
- 人脉资源
- 项目信息
- 需求信息
- 社群 / 二维码入口

## 7.2 为什么必须脱敏

原因有三层：

### 第一层：保护资源方

不能把真实可联系的人、真实组织、真实渠道无条件公开抛到网上。

### 第二层：保护平台

平台必须避免看起来像公开兜售敏感资源、关系资源或不受控撮合。

### 第三层：保护业务价值

如果所有信息都直接摊开，平台就失去了会员价值、桥接价值和居间价值。

## 7.3 脱敏展示的基本规则

公开页可以展示：

- 地区
- 行业
- 大类方向
- 资源属性
- 摘要描述
- 是否可桥接
- 是否支持邮箱触达
- 是否需要会员解锁

公开页不直接展示：

- 全姓名
- 直接联系方式
- 精确内部身份
- 可被立即绕开的关键路径
- 完整二维码原图
- 会导致资源方被瞬时打穿的信息

## 7.4 深层信息的三种触达方式

平台对深层信息的开放方式统一为三种：

### A. 会员解锁

适用于低到中等敏感度信息。

### B. 邮箱触达

适用于不宜页面直接展示、但可以通过更稳妥方式触达的信息。

### C. 人工桥接

适用于高敏感度、高价值、需要更明确边界和风控保护的信息。

## 7.5 会员解锁逻辑

既有方案中的会员逻辑保留，但需要纳入统一业务语言。

当前可作为母稿暂定规则：

- 免费用户：浏览脱敏摘要
- 普通会员：解锁有限数量完整信息
- 高级会员：解锁更多完整信息
- 尊享会员：解锁更高权限和更多人工桥接机会

当前既有定价方案可暂定为：

- 普通会员：99 元 / 年
- 高级会员：199 元 / 年
- 尊享会员：299 元 / 年

当前既有查看规则可暂定为：

- 普通会员每日限 2 条完整信息
- 高级 / 尊享会员不限次数

后续可以微调价格和权益，但母逻辑不变：

`免费摘要吸引注册，会员解锁承接价值，邮箱与人工桥接承接高敏信息。`

## 7.6 邮箱解锁的角色

邮箱触达不是临时补丁，而是平台非常关键的中间层。

邮箱触达适合用于：

- 避免深层信息直接在站内公开
- 避免一次性页面暴露过深细节
- 给资源方和平台都保留缓冲层
- 为后续人工桥接留空间

因此，邮箱不是可有可无，而是平台“克制表达、合规承接、保护资源边界”的重要工具。

## 8. 居间 / 引荐 / 桥接能力总纲

## 8.1 定位原则

平台确实有居间功能模块，但必须合规、克制、正气。

它不是：

- 违法请托
- 灰产撮合
- 不透明利益分配工具
- 帮人绕规则的工具

它应被定义为：

`审核前置 + 脱敏展示 + 规则说明 + 邮箱触达 + 人工桥接 + 协同留痕 + 风控保护`

## 8.2 模块名称建议

对外建议不把“防跳单”“控局”写得过直。

平台内部和产品层最合适的模块名称建议为：

`项目协同护航`

这个名称能传达：

- 项目在推进
- 多方在协同
- 平台在护航
- 不显得江湖
- 更容易避嫌

如果要放在一级主菜单上，建议优先使用更短、更稳、更高级的表达。

一级主菜单名称建议按优先级考虑：

1. `协同护航`
2. `合作护航`
3. `桥接协同`
4. `协同桥接`

当前首选建议：

`协同护航`

原因是它最适合做一级菜单，字数短，辨识度高，既能涵盖居间、引荐、桥接、推进，又不会把“居间”二字说得过硬。

## 8.3 为什么平台需要这个模块

现实里很多项目发起人、引荐人、协同人都面临同类难题：

- 信息一出去就可能被绕开
- 自己牵了线，但过程贡献没有被记录
- 项目推进中角色边界不清晰
- 合作各方接触后，中间协调者容易被抹掉
- 没有留痕工具，就很难维持秩序

所以这个模块不是“控局工具”，而是：

`为资源发起、引荐桥接、过程推进和边界保护提供一个合规、可留痕、可风控的协同工具。`

## 8.4 一级 / 二级引荐人定义

这是本平台引荐逻辑中的关键概念，必须统一。

### 一级引荐人

最先接住原始资源或原始需求的人。

也就是说，一级引荐人是最早把：

- 资源接进来的人
- 需求接进来的人
- 原始项目线索接进来的人

他是第一承接点，也是最早形成初始连接关系的人。

### 二级引荐人

在一级引荐人之后，继续承接转介绍、继续桥接、继续撮合的人。

也就是说，二级引荐人并非原始资源第一持有人，但在后续推进中承担了继续连接、再桥接、再促成的作用。

## 8.5 引荐逻辑的产品意义

定义一级 / 二级引荐人的意义不只是分润，而是为了：

- 还原贡献链
- 留存桥接链
- 保护协同链
- 让平台知道谁最先接住线索，谁继续推进了线索

这是后续居间、协同、分润、风控的基础。

## 8.6 居间 / 协同模块未来可承载的能力

当前阶段可以先按能力方向理解，不必一次全做完：

- 发起协同
- 邀请协同人
- 记录原始资源 / 需求来源
- 记录一级 / 二级引荐关系
- 留存过程备注
- 标记阶段进展
- 上传保密前置材料
- 说明履约边界
- 触发人工桥接
- 形成基础留痕

## 8.7 协同护航模块的核心交互逻辑

这一模块不能做成普通表单页，而应做成“角色识别 + 链路生成 + 层层接力 + 最终闭环”的产品。

它的核心逻辑是：

1. 某个人先发起一条协同信息
2. 系统引导他选择自己的身份
3. 系统生成一个可分享的专属链接
4. 他把链接发给自己的朋友或下一层承接人
5. 下一层承接人继续确认身份并补链
6. 链路不断延长
7. 最终逐步把甲方与乙方两边完整链条搭起来

也就是说，这个模块的核心不是“上传资料”，而是：

`让一条原本散落在人与人之间的居间链，被系统一步步搭成一条完整协同链。`

## 8.8 身份识别机制

进入协同护航模块后，用户不一定天然知道自己属于哪一级角色，因此系统不能让用户直接随意乱填，而应做成引导式身份识别。

建议系统先让用户通过选择题判断自己身份，例如：

- 你是否直接接近甲方拍板人
- 你是否直接接近乙方拍板人
- 你是否是项目发起方本人
- 你是否只是继续转介绍、继续桥接的人

系统最终将角色归入以下几类：

- 甲方拍板人
- 乙方拍板人
- 甲方一级引荐人
- 乙方一级引荐人
- 二级居间人 / 二级引荐人

这里要特别强调：

- 一级引荐人，不是“谁最活跃”，而是“谁离甲方或乙方拍板权最近”
- 二级引荐人，是一级之后继续承接、继续桥接、继续撮合的人

## 8.9 U 型链路的视觉表达

这个模块必须形象化，不能只做表格和文字。

最适合的主视觉结构，就是你提到的：

`U 型链子 / U 型桥接动画`

其视觉含义如下：

- 左岸代表甲方
- 右岸代表乙方
- 左右两岸各有一个核心代表人物
- 靠近两岸拍板人的那两个关键节点，分别代表甲乙双方的一级引荐人
- 中间所有承接、转介绍、再桥接的人，统一组成 U 型链中段，代表二级引荐人群体

这不是普通装饰动画，而是在用视觉把规则讲明白：

`越靠近两岸拍板端的人，层级越关键；中间链条越长，说明桥接越复杂。`

## 8.10 动画与链路完成状态

这个模块的视觉建议做成一种“链路逐步点亮”的感觉。

例如：

- 发起时，只亮起一个起点
- 有新的人通过链接加入，链条就多亮一节
- 甲方侧形成一级引荐关系时，左岸近端节点高亮
- 乙方侧形成一级引荐关系时，右岸近端节点高亮
- 当甲乙两岸都被接通时，整条 U 型桥发光或闭合

这样用户一看就明白：

- 链条还缺哪一段
- 哪一边还没接到关键人
- 当前是否已经完成真正意义上的“搭桥”

## 8.11 分享链接机制

这个模块必须有“发起后生成链接”的机制，因为现实中的桥接不是一个人一次性填完的，而是层层转发、层层确认形成的。

建议逻辑如下：

- 发起人创建一条协同链
- 系统生成一条专属链接
- 发起人发送给下一位承接人
- 下一位承接人进入后，只看到与自己相关的简明信息
- 其确认角色后，系统再允许其继续向下一位转发

这样，链路就不是后台硬录入，而是自然从人际传播中长出来。

## 8.12 信息可见范围

这部分非常关键。

你提到的要求本质上是：

- 协同参与者看到的信息，一律公开、简明、克制
- 甲乙双方不应该知道二级居间人到底有多少个
- 如果中间人过多，被甲乙双方看见，往往会导致他们不愿继续参与

因此权限设计建议如下：

### 甲方 / 乙方拍板人可见

- 项目简明摘要
- 当前是否已形成有效桥接
- 与自己直接相关的近端关键节点
- 参考合同 / 模版 / 协议建议
- 当前进度状态

默认不直接可见：

- 全部二级引荐人数
- 完整中间链条细节
- 所有中间人的身份明细

### 一级引荐人可见

- 自己所在一侧的关键链路
- 基础进展信息
- 必要的模板与规则说明
- 与自己相邻的上下游节点

### 二级引荐人可见

- 公开简明项目摘要
- 自己所处链路位置
- 与自己直接相邻的必要节点信息
- 是否继续补链 / 转发的操作入口

默认不应看到：

- 甲乙双方所有深层资料
- 不必要的完整链条结构
- 会导致越权绕开的敏感信息

### 平台后台可见

- 完整链路
- 全部角色
- 时间顺序
- 分享记录
- 风险提示

## 8.13 合同与模板的植入方式

这个模块不应只是“一个动画”，还要把你目录里的规则、合同、建议模板真正变成工具。

建议在协同护航模块中植入三类材料：

- 参考合同
- 建议模板
- 风险提示与注意事项

使用方式建议为：

- 平台提供“推荐模板”
- 甲乙双方可在此基础上修改
- 平台保留标准版、保密脱敏版、简化版

这样设计的意义是：

- 给不会操作的居间业务人一个现成抓手
- 让规则和模板变成可执行工具
- 降低临场协同时的混乱感

## 8.14 协同护航模块的真实价值

这个模块的真实价值，不是把“居间”讲得更硬，而是把很多不会操作的人，用系统化方式带进一个更稳的流程里。

它要解决的是：

- 很多居间业务人不会搭规则
- 不知道自己到底属于哪一级
- 不知道怎么层层传递关系
- 不知道如何既保护边界又推动合作
- 不知道该用什么合同和模板

所以这个模块最终做的事，是：

`把原本只存在于人情世故和口口相传里的居间操作，变成一个可视化、可引导、可留痕、可避嫌、可协同的系统化流程。`

## 9. 合规与风险边界

## 9.1 对外表达总原则

平台必须懂中国式资源连接逻辑，但公开表达要克制、合规、避嫌。

可以讲：

- 资源对接
- 项目协同
- 合规桥接
- 人工辅助对接
- 脱敏展示
- 风控保护
- 履约边界
- 保密前置

不宜直白讲：

- 搞关系
- 内部渠道
- 打点
- 防跳单
- 控局
- 锁人
- 灰色中介

## 9.2 平台必须体现的风险控制能力

平台的合规边界必须具体落到产品结构中，而不是只写在声明里。

应逐步体现：

- 内容审核
- 脱敏展示
- 人工桥接
- 保密边界提示
- 风险提示
- 举报与下架
- 履约边界说明
- 过程留痕

## 9.3 风险边界的真实作用

风险边界不是为了把平台做得“死”，而是为了：

- 给资源方安全感
- 给平台合规缓冲
- 给引荐人和协同人边界保护
- 给会员体系价值支撑

## 10. 中文化 / SEO / AI 推广总纲

## 10.1 中文化原则

平台是面向中文互联网传播的产品，必须采用全中文、强本土化、易理解的表达体系。

要求包括：

- 全站按钮、标题、导航、提示语尽量全中文
- 不做中英文混杂表达
- 文案要符合中文商务表达习惯
- 既要清楚，又不能太直白
- 适合搜索抓取，也适合 AI 理解和改写传播

## 10.2 SEO 原则

站内内容必须天然适合中文搜索引擎收录。

要求包括：

- 页面标题含中文核心关键词
- 页面描述含清晰业务价值
- URL 简洁，便于中文语义理解
- 图片具备中文 alt
- 核心栏目可被站点地图收录
- 后台、隐私、敏感页不参与公开收录

核心关键词不应堆砌，但应稳定围绕这些方向：

- 资源对接
- 项目协同
- 人脉资源
- 行业社群
- 会员解锁
- 合规桥接

## 10.3 AI 推广适配原则

平台要方便 AI 读懂，也方便 AI 拿去二次生成推广文案。

因此必须做到：

- 核心信息结构化
- 会员价格清晰
- 核心能力清晰
- 核心规则清晰
- 合规边界清晰

首页与会员页应逐步沉淀出可直接被 AI 抽取的几类内容：

- 一句话定位
- 核心优势列表
- 会员权益列表
- 风险边界说明
- 常见问题说明

## 10.4 短视频与搜索的关系

短视频平台是前端入口，搜索引擎是中长期承接渠道，网站是统一承接中枢。

关系应理解为：

- 短视频负责激发兴趣和引发点击
- 搜索负责沉淀自然流量
- 网站负责建立信任、转化会员、引导桥接

## 12. 当前开发总原则

当前开发不能再拆着推进，而应按这份母稿收口。

首页、模块、会员、后台、文案、SEO、AI 推广都必须基于同一套产品理解。

当前阶段最重要的不是继续堆页面，而是先把这几件事统一：

1. 首页必须保留 3D 地球，但不走空概念路线

8. 整体表达必须中文化、SEO 友好、AI 推广友好
